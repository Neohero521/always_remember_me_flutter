### 3. 当前 3-Tab 布局问题
```
Tab1: 书架 (但包含了首页快捷操作)
Tab2: 续写 (核心功能但藏在中间)
```

- 阅读入口藏在 AppBar 菜单里，不直观
- 设置散落在各个 PopupMenu 里

## 新 UI 方案（推荐）

### 方案：**单Tab + Drawer 方案**（最适合「续写工具」定位）

```
App 启动 → WriteScreen（续写核心，作为首页）
          ↓
    顶部 AppBar: 书名 + 阅读按钮 + 设置
          ↓
    状态概览卡片（当前小说、章节数、图谱数）
          ↓
    续写操作面板（选章节 → 续写 → 预览 → 加入链条）
          ↓
    续写链条列表（最近续写结果）
          ↓
    Drawer 侧边栏: 书架 | 章节管理 | 图谱管理 | 导入/导出
```

### 理由
1. **这个 App 是「续写工具」**，不是「书架管理工具」
2. 用户核心路径：`选书` → `选章节` → `续写` → `看结果`
3. 书架、章节管理、图谱是「辅助功能」，不应该和核心续写功能平起平坐
4. 单 Tab 把所有核心操作集中在一个屏幕，减少认知负担

### 导航结构
| 入口 | 位置 | 说明 |
|------|------|------|
| 续写（核心） | 首页 Tab，始终可见 | |
| 阅读 | AppBar 右侧按钮 | 点击即开阅读器 |
| 书架 | Drawer 第一项 | 选书、切书 |
| 章节管理 | Drawer 第二项 | 图谱生成、查看 |
| 图谱总览 | Drawer 第三项 | 全局图谱查看 |
| 设置 | AppBar 溢出菜单 | 续写参数、主题等 |

## 产品 PRD（产品需求定义）

### 一、产品定位
**Always Remember Me** 是一款**小说续写辅助工具**，通过 AI 知识图谱技术帮助作者保持人物设定一致性、伏笔连贯性，实现高质量续写。

### 二、目标用户
- 网络小说作者
- 有续写需求的小说爱好者
- 需要保持长篇剧情一致性的创作者

### 三、核心功能

#### 3.1 小说导入与管理
- 支持导入 TXT 文件，自动解析章节
- 多本书籍管理（书架）
- 当前小说切换

#### 3.2 AI 续写
- 选择基准章节作为续写起点
- 可编辑基准章节内容后重写
- 前置校验：人设红线检查、禁区规则、伏笔引用
- 续写预览 + 质量评估
- 续写链条：管理多个续写结果，支持"基于续写继续写"
- 批量续写：自动从起始章节向后续写全部章节，支持重试和停止

#### 3.3 知识图谱
- 单章图谱生成：人物、世界观、剧情线、文风特点
- 批量图谱生成：一次性为所有章节生成图谱
- 图谱合并：分批合并 + 全局合并
- 全局图谱查看
- 图谱合规性校验

#### 3.4 图谱导入/导出
- JSON 格式导出，方便备份和迁移
- JSON 格式导入，恢复数据

### 五、非功能需求
- 启动时间 < 3秒（数据加载时显示动画）
- 图谱生成可中断（停止按钮有效）
- 批量续写可中断（停止按钮有效）
- 防空回检测：续写字数 < 50 字自动标记失败
- JSON 导入需做安全校验（防止 XSS/注入）

### 六、验收标准
1. 单 Tab 首页可以完成所有核心操作（续写、查看链条）
2. Drawer 可以切换书籍、进入章节管理、查看图谱
3. 所有现有功能（续写、图谱生成/合并、导入导出）在新 UI 下正常工作
4. 无功能 regression
5. APK 可以正常安装运行
