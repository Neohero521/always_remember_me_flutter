# Always Remember Me v4.0 深度重构计划
> 项目总指挥输出 · 2026-04-11

## 一、现状问题分析

### 1.1 代码结构问题

| 问题 | 详情 | 严重度 |
|------|------|--------|
| **死代码** | `home_screen.dart`（554行）从未被引用，完全浪费 | 🔴 高 |
| **界面过载** | BookshelfScreen 单文件承载：书架 + 快捷操作 + 状态概览 + 图谱管理 + 导入导出 | 🔴 高 |
| **Tab滥用** | WriteScreen 的3-Tab布局：续写（核心）藏中间，链条和图谱在Tab2/3 | 🔴 高 |
| **功能重复** | 续写入口、图谱生成/合并入口散落在 BookshelfScreen + WriteScreen + ChaptersScreen 多处 | 🟡 中 |
| **导航混乱** | 首页（home_screen.dart）写了不用，用户实际入口是书架 | 🔴 高 |
| **GraphStatusTab 无用** | WriteScreen 的图谱状态Tab内容贫乏，真正的图谱在 GraphViewerScreen | 🟡 中 |
| **Drawer 冗余** | Drawer 内容过多（9+项），且与 BottomNavigation 重叠 | 🟡 中 |

### 1.2 UI 具体问题

**WriteTab（最核心页面）问题：**
- 信息密度过高：基准章节选择 → 前置校验 → 续写预览 → 质量评估 → 操作按钮 → 批量续写，全部堆在一个 ListView
- 视觉层次不清：校验结果（rarely needed）和续写操作（高频）在同一层级
- 批量续写对话框太复杂：选择章节列表嵌入 AlertDialog，体验差

**BookshelfScreen 问题：**
- `_QuickActionsSection` 做了「首页」的工作但命名是「书架」
- 状态卡片 + 5个快捷操作卡片 + 书架列表，全塞在一个可滚动页面
- 没有书的空状态和选了书后的状态，切换时页面跳变

**ChaptersScreen 问题：**
- 作为Tab3使用频率最低但占固定位置
- 图谱生成进度状态没有实时反馈
- 多选批量操作没有视觉引导

## 二、v4.0 重构方案

### 2.1 新导航结构：**BottomNav + Drawer**

```
┌─────────────────────────────────────┐
│  ✍️ 续写   📖 阅读   📚 书架         │  ← 底部导航（3项）
├─────────────────────────────────────┤
│                                     │
│         当前 Tab 内容                │
│                                     │
├─────────────────────────────────────┤
│ ☰  →  Drawer: 章节管理 | 图谱总览 | 设置  │
└─────────────────────────────────────┘
```

**BottomNav 三项定位：**
| Tab | 功能 | 优先级 |
|-----|------|--------|
| ✍️ 续写 | 续写操作 + 续写链条（整合原 WriteScreen） | 最高 |
| 📖 阅读 | 舒适阅读器（独立Tab，频繁使用） | 高 |
| 📚 书架 | 书籍管理（导入、切书、删除） | 中 |

**Drawer（从左滑出）五项：**
| 项目 | 功能 |
|------|------|
| 章节管理 | 图谱生成、批量生成、章节状态 |
| 全局图谱 | 合并后的完整知识图谱查看 |
| 图谱导入/导出 | JSON 格式管理 |
| 续写参数设置 | 前置校验开关、批量续写设置 |
| 设置 | App 设置、主题、关于 |

### 2.2 核心页面重新设计

#### 2.2.1 续写页（WriteScreen v4）

**Layout：单页 + 分区，不嵌套 Tab**

```
┌─────────────────────────────────────┐
│ AppBar: [☰] 当前书籍    [📖阅读] [⋮] │
├─────────────────────────────────────┤
│  ┌──────────────────────────────┐   │
│  │ 📊 状态卡片（可折叠）          │   │  ← 默认展开，点击折叠
│  │ 12章节 · 8图谱 · 3续写 · ✅合并 │   │
│  └──────────────────────────────┘   │
├─────────────────────────────────────┤
│  续写基准章节：                      │
│  ┌──────────────────────────────┐   │
│  │ 第3章 夜色降临          [▼]  │   │  ← Dropdown选章节
│  │ ✅ 有图谱  ·  约 2000 字      │   │
│  └──────────────────────────────┘   │
│                                     │
│  [🚀 开始续写]                      │  ← 主操作按钮，大而醒目
│  [⚡ 一键批量续写]  [📋导入小说]     │  ← 次要操作
│                                     │
├─ 续写结果（生成后展开）───────────────┤
│  ┌──────────────────────────────┐   │
│  │ 续写内容（约 1200 字）         │   │  ← 默认折叠，生成后自动展开
│  │ ━━━━━━━━━━━━━━━━━━━━━━━━━━   │   │
│  │ 第3章续写...（正文预览）       │   │
│  │                    [📋复制]   │   │
│  ├──────────────────────────────┤   │
│  │ 质量评估：78分 ✅ 合格         │   │
│  │ 前置校验：人设红线✓ 禁区✓ 伏笔3 │   │
│  └──────────────────────────────┘   │
│                                     │
│  [✅ 加入续写链条]  [➡️ 基于此继续]   │
├─────────────────────────────────────┤
│  ✍️ 续写链条（3条）        [清空]   │  ← 可折叠列表
│  ┌──────────────────────────────┐   │
│  │ 🔗 续写#1  ·  1200字         │   │
│  │ [→继续] [📋] [🗑]             │   │
│  └──────────────────────────────┘   │
└─────────────────────────────────────┘
```

**设计原则：**
1. **状态卡片可折叠** — 减少视觉噪音，需要时展开
2. **续写结果默认隐藏** — 未生成时不显示，生成后自动展开
3. **操作按钮大而少** — 主按钮（开始续写）最醒目，次要操作（批量、导入）较小
4. **链条列表折叠** — 空时不占位，有内容时默认展开
5. **前置校验通过时隐藏详情** — 只在有问题时才展开

#### 2.2.2 阅读页（ReaderScreen v4）

保持现有功能，做以下精简：
- 移除重复的 AppBar（页面本身已有）
- 底部控制栏：进度条 + 字号，减少不必要的操作
- 章节列表通过下滑或点击中间区域唤出

#### 2.2.3 书架页（BookshelfScreen v4）

**Layout：简化为书籍列表**

```
┌─────────────────────────────────────┐
│ AppBar: [☰] 我的书架    [📥导入] [⋮] │
├─────────────────────────────────────┤
│  （选中书籍后显示快捷操作条）          │  ← 选书后才显示，有书才能操作
│  [📖阅读] [✍️续写] [🌲图谱]          │
├─────────────────────────────────────┤
│  📖 我的书架                         │
│  ┌──────────────────────────────┐   │
│  │ [封] 《书名》  12章  上次阅读  │   │  ← 当前书籍高亮
│  │ [面] [续写] [删除]  阅读进度    │   │
│  └──────────────────────────────┘   │
│  ┌──────────────────────────────┐   │
│  │ [封] 《书名2》  8章   3天前    │   │
│  └──────────────────────────────┘   │
└─────────────────────────────────────┘
```

**设计原则：**
1. **一本书一行** — 封面 + 书名 + 章节数 + 上次阅读 + 操作
2. **当前书籍特殊标记** — 左边框高亮 + 「当前」标签
3. **操作简洁** — 续写、阅读、删除，不做图谱管理（交给 Drawer）
4. **空状态引导清晰** — 提示导入小说

#### 2.2.4 Drawer 侧边栏（v4精简版）

```
┌──────────────────────────┐
│ 🐋 Always Remember Me    │
│ 当前：《小说名》           │
├──────────────────────────┤
│ 📖 章节管理        →      │  ← 章节列表 + 图谱生成入口
│ 🌲 全局图谱        →      │  ← 合并后的知识图谱
│ 📤 导入/导出图谱   →      │
│ ⚙️ 续写参数        →      │
│ ───────────────────────── │
│ 📱 设置            →      │
│ ℹ️ 关于            →      │
└──────────────────────────┘
```

## 四、优先级和时间线

### Phase 1：基础框架（3-4天）
1. 实现 BottomNavigation 骨架（WriteScreen + ReaderScreen + BookshelfScreen）
2. 实现精简版 Drawer（章节管理、全局图谱、设置入口）
3. 删除死代码（home_screen.dart、GraphStatusTab、AppShell）
4. 确保导航可工作

### Phase 2：WriteScreen 重构（4-5天）
1. 重写 WriteTab 布局（分区 + 可折叠卡片）
2. 实现续写结果自动展开逻辑
3. 简化批量续写为 BottomSheet
4. 续写链条移入 WriteScreen 底部
5. 质量评估/前置校验折叠显示

### Phase 3：书架简化 + 阅读器（2-3天）
1. 简化 BookshelfScreen（去除 QuickActionsSection 逻辑）
2. 快捷操作移至 Drawer
3. 阅读器保持不变（已足够好）

### Phase 4：Drawer 功能完善（2-3天）
1. 章节管理（含图谱生成、批量生成、状态显示）
2. 全局图谱查看（复用现有 GraphViewerScreen）
3. 导入/导出图谱
4. 续写参数设置页

### Phase 5：测试 + 打磨（2-3天）
1. 功能回归测试
2. UI 细节打磨
3. 打包验证
